计算机组成原理
发布于 2018-05-14   更新于 2019-06-30 笔记

计算机层次结构

计算机分类和发展方向

  • 指令和数据流

    • 单指令流&单数据流SISD（冯诺依曼体系）
    • 单指令流&多数据流SIMD（阵列处理器、向量处理器）
    • 多指令流&单数据流MISD（不存在）
    • 多指令流&多数据流MIMD（多处理器、多计算机）

  • 发展方向

    • 便携
    • 处理能力

计算机功能部件

  • 组成（软件和硬件在逻辑上是等效的）

    • 硬件系统

      • CPU（运算器+控制器）
        CPU+主存储器=主机
      • 外设：输入设备、输出设备、辅助存储器
      • 输入设备
      • 输出设备
      • 运算器：处理原始数据、把计算结果输出到输出设备
      • 控制器：控制总线是双向的，控制输入、输出设备、控制器、存储器
      • 存储器：存储数据、指令（计算步骤或程序），方便运算器读取输出数据、指令。

    • 软件系统

软硬件联系

软件硬件无严格界限，互相依存，协同发展。

CPU

  • 运算器
    • MQ 乘商寄存器（Multiple-Quotient Register）
    • ACC 累加器（Accumulator）
    • ALU 算术逻辑单元（Arithmetic and Logic Unit）组合逻辑
    • X 通用寄存器
    • PSW 程序状态字寄存器（Program Status Word）
  • 控制器
    • CU 控制单元（Control Unit） 分析指令，给出控制信号。时序逻辑
    • IR 指令寄存器（Instruction Register）存放当前执行的指令
    • PC Program Counter 存放指令地址，有自动加1功能。

取数指令

取数分为三个阶段，取指令->分析指令->执行指令

CPU区分指令和数据的依据：指令周期的不同阶段。取指令阶段，取出的放到IR中，在执行指令阶段当作数据放入相应寄存器中。

存储器

存储器存取数据由时序控制逻辑控制

存储体实现存储模式有很多种，工作模式类似。

存储器数据寄存器（MAR）、存储器地址寄存器（MBR）在设计时属于主存，但是在实现过程中时做在CPU中。

译码器

n位地址，对应2的n次方个存储单元

冯诺依曼体系

  • 计算机硬件系统由运算器、存储器、控制器、输入设备和输出设备5大部件组成。
  • 指令和数据以同等地位存于存储器内，并可按地址寻访。
  • 指令和数据均用二进制代码表示。
  • 指令由操作码和地址吗组成，操作码用来表示操作的性质，地址吗用来表示操作数在存储器中的位置。
  • 指令在存储器内按顺序存放。通常，指令是顺序执行的，在特定条件下，可根据运算结果或根据设定的条件改变执行顺序。
  • 早期的冯诺依曼机以运算器为中心，I/O设备通过运算器和存储器传送数据。

计算机性能指标

容量

总容量=存储单元个数x存储字长 bit

系统所能支持的最大容量是2的地址位数次方x存储字长
